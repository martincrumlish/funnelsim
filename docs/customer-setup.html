FunnelSim Quick Setup

Deploy your own funnel simulation platform in ~30 minutes

0
What You'll Need
  • Supabase account - Create free account
  • Stripe account - Create account
  • GitHub account - For Vercel deployment

No coding or command line required!

1
Deploy to Vercel
~2 min

Click the button below to deploy your app to Vercel:

Deploy with Vercel
  1. Sign in with GitHub
  2. Click "Create" to fork the repository
  3. Skip environment variables for now (we'll add them later)
  4. Wait for deployment to complete
Note: The app won't work yet - we need to set up Supabase first.
2
Create Supabase Project
~3 min
  1. Go to Supabase Dashboard
  2. Click "New Project"
  3. Enter a project name (e.g., "my-funnelsim")
  4. Set a strong database password (save this somewhere safe!)
  5. Select a region close to your users
  6. Click "Create new project"
  7. Wait 2-3 minutes for setup to complete
3
Set Up Database
~2 min
  1. In your Supabase project, go to SQL Editor (left sidebar)
  2. Click "New query"
  3. Copy the entire contents of combined-migrations.sql
  4. Paste into the SQL editor
  5. Click "Run" (or press Ctrl/Cmd+Enter)
Success! You should see "Success. No rows returned" - this is correct!
4
Verify Database Setup
~30 sec

The migrations automatically created:

  • All database tables (funnels, profiles, subscriptions, etc.)
  • Storage bucket funnel-logos with proper policies
  • Row Level Security policies

You can verify in Supabase Dashboard > Table Editor and Storage.

5
Configure Stripe
~10 min

A. Get API Keys

  1. Go to Stripe Dashboard > Developers > API keys
  2. Copy your Publishable key (starts with pk_test_)
  3. Copy your Secret key (starts with sk_test_) - you'll need this for Step 6

B. Create Products

Go to Products and create:

Product Prices to Create
Pro Plan Monthly: $29/month (recurring)
Yearly: $290/year (recurring)
Lifetime: $499 one-time (optional)
Enterprise Plan Monthly: $99/month (recurring)
Yearly: $990/year (recurring)
Lifetime: $1,499 one-time (optional)

Save all the Price IDs (they start with price_)

C. Create Webhook

  1. Go to Developers > Webhooks
  2. Click "Add endpoint"
  3. Endpoint URL: https://YOUR_PROJECT_ID.supabase.co/functions/v1/stripe-webhook
    (Replace YOUR_PROJECT_ID with your Supabase project ID from Settings > General)
  4. Events to listen for:
    • checkout.session.completed
    • customer.subscription.updated
    • customer.subscription.deleted
    • charge.refunded
    • invoice.payment_failed
  5. Click "Add endpoint"
  6. Copy the Signing secret (starts with whsec_)
6
Deploy Edge Functions
~10 min

Edge functions handle Stripe payments and admin operations. You have two options:

Option A: Using Supabase CLI (Recommended)

  1. Install Supabase CLI
  2. Clone your forked repo locally
  3. Run these commands: 
    # Login to Supabase
supabase login

# Link to your project
supabase link --project-ref YOUR_PROJECT_ID

# Set secrets
supabase secrets set STRIPE_SECRET_KEY=sk_test_your_key
supabase secrets set STRIPE_WEBHOOK_SECRET=whsec_your_secret

# Deploy all functions
supabase functions deploy create-checkout-session --no-verify-jwt
supabase functions deploy create-portal-session --no-verify-jwt
supabase functions deploy stripe-webhook --no-verify-jwt
supabase functions deploy retrieve-checkout-session --no-verify-jwt
supabase functions deploy link-pending-subscription --no-verify-jwt
supabase functions deploy admin-create-user
supabase functions deploy admin-delete-user
supabase functions deploy admin-reset-password

Option B: Manual (No CLI needed)

  1. Go to your Supabase Dashboard > Edge Functions
  2. Click "Create a new function"
  3. For each function, copy the code from the GitHub repo's supabase/functions/ folder
  4. Go to Project Settings > Edge Functions to add secrets:
    • STRIPE_SECRET_KEY = your Stripe secret key
    • STRIPE_WEBHOOK_SECRET = your webhook signing secret
Important: Functions that handle Stripe webhooks must be deployed with --no-verify-jwt to accept requests from Stripe.
7
Configure Vercel Environment Variables
~3 min

Go to your Vercel project > Settings > Environment Variables and add:

Variable Where to Find It
VITE_SUPABASE_PROJECT_ID Supabase > Settings > General
VITE_SUPABASE_PUBLISHABLE_KEY Supabase > Settings > API > anon/public key
VITE_SUPABASE_URL Supabase > Settings > API > Project URL
VITE_STRIPE_PUBLISHABLE_KEY Stripe > Developers > API keys > Publishable key

After adding variables, click "Redeploy" in Vercel (Deployments tab > ... menu > Redeploy)

8
Create Admin Account
~2 min
  1. Visit your deployed app and sign up for an account
  2. Go to Supabase > Authentication > Users
  3. Click on your user and copy the User UID
  4. Go to Table Editor > admin_users
  5. Click "Insert row"
  6. Set user_id to your UID, is_admin to true
  7. Save

You can now access /admin in your app!

9
Link Stripe Products
~2 min
  1. Go to /admin/products in your app
  2. Click Edit on "Pro"
  3. Add your Stripe Price IDs:
    • stripe_price_id_monthly: price_xxx
    • stripe_price_id_yearly: price_xxx
    • stripe_price_id_lifetime: price_xxx (optional)
  4. Save and repeat for "Enterprise"

You're Done!

Your FunnelSim instance is now fully configured.

Test a checkout using Stripe test card: 4242 4242 4242 4242

Setup Checklist
  • Vercel deployment complete
  • Supabase project created
  • Database migrations applied (creates tables + storage bucket)
  • Edge functions deployed
  • Environment variables set in Vercel
  • Admin account created
  • Stripe products created (optional - can add later)
  • Stripe webhook configured (optional - can add later)
  • Stripe products linked in admin (optional - can add later)
+

Go to /admin/settings to customize:

  • Brand name and tagline
  • Logo (light and dark versions)
  • Primary color
  • Landing page content (hero, features, testimonials, FAQ)
